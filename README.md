# NetCDF Viewer

Java desktop application for opening unstructured-triangle NetCDF files, inspecting metadata, and rendering planar scalar fields with optional layer switching.

## Features

- Open `.nc` files from the File menu or by drag-and-drop
- Read dimensions, variables, and global attributes with NetCDF-Java 5.x
- Detect coordinate variables and triangle connectivity arrays automatically
- Render planar triangle-filled scalar maps on a JavaFX canvas
- Switch layers for supported `[layer, node]` variables
- Show color bar and current min/max range
- Export the current visualization to PNG
- Package the application into a Windows `.exe` with `jpackage`

## Environment

- JDK 17+ recommended
- Maven 3.9+
- Windows for `.exe` packaging

The project is currently built and tested on Java 21 and Maven 3.9.

## Build And Test

Run the test suite:

```powershell
mvn test
```

Create the application jar and copy runtime dependencies:

```powershell
mvn clean package
```

Run the JavaFX application during development:

```powershell
mvn javafx:run
```

Do not start the plain jar with `java -jar target\\netcdf-viewer-1.0.2.jar`. That jar is not a self-contained JavaFX distribution. Use `mvn javafx:run` for development, or use the packaged Windows launcher generated by `jpackage`.

## Package As EXE

Run the packaging script:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\package-exe.ps1
```

Expected output:

- application input directory under `target\package-input`
- generated installer under `target\installer`
- installer filename `NetCDFViewer-1.0.2.exe`

## Notes

- Connectivity indices are normalized automatically from either 0-based or 1-based input.
- Coordinate axis variables such as `lon`, `lat`, `x`, `y`, `siglay`, and similar axis-like arrays are excluded from the plottable attribute list.
- If a file opens but lacks a compatible coordinate pair or triangle connectivity array, metadata is still shown and the UI displays a friendly visualization warning instead of crashing.
