# NetCDF Viewer

一款基于 JavaFX 的 NetCDF 桌面可视化工具，支持非结构化三角网与标准格网 `.nc` 文件的读取、二维平面填色、单点查询、多数据集切换、海岸线叠加、波向箭头叠加、海流流线叠加与 PNG 导出。

## 项目定位

本项目面向海洋、水文、环境与数值模拟数据处理场景，重点解决以下问题：

- 读取非结构化三角网与标准格网 NetCDF 数据
- 自动识别坐标变量、三角形连接关系、结构化坐标基准与兼容变量对
- 对单层与多层属性变量进行平面可视化与叠加分析
- 在 Windows 环境下生成可直接分发的安装包

## 项目亮点

- 同时覆盖非结构化三角网与标准格网场景
- 自动识别坐标变量、连接关系、结构化坐标基准、可平面化属性变量、波浪变量对和流场速度变量对
- 同时支持节点中心与单元中心属性变量显示
- 支持多层变量深度切换、平面图单点查询、结构化坐标轴切换与多数据集管理
- 支持 GeoJSON / shapefile 海岸线叠加，并内置默认海岸线资源
- 支持波向箭头与海流动态流线叠加，三角网和标准格网都可复用同一分析入口
- 支持导出当前视图为 PNG，便于报告、汇报与结果归档
- 支持打包为无需额外安装 JRE 的 Windows 安装程序

## 主要功能

- 支持通过菜单或拖拽方式打开 `.nc` 文件
- 支持在已加载一个数据集后继续追加加载其他 `.nc` 数据，并切换或删除已加载数据集
- 展示变量、维度、全局属性、坐标变量、坐标基准和连接变量信息
- 支持单层变量与多层变量的平面渲染
- 支持深度层切换
- 支持结构化数据按变量自动匹配兼容横纵坐标，并允许用户切换候选坐标轴
- 支持平面图单点查询，三角网节点中心变量返回插值值，单元中心变量返回命中单元原始值；标准格网返回命中节点或单元值
- 支持颜色映射与当前层最小值 / 最大值显示
- 支持 GeoJSON / shapefile 海岸线叠加与内置海岸线恢复
- 支持自动识别 `wdir` / `wlen` 波场变量并叠加波向箭头
- 支持自动识别 `u` / `v` 或 `ua` / `va` 流场变量，并在三角网或标准格网上叠加动态海流流线
- 支持导出当前可视化结果为 PNG 图片
- 支持打包为独立 Windows 安装程序 `.exe`

## 快速开始

### 1. 运行测试

```powershell
mvn -q test
```

公开仓库或 CI 环境如果没有本地 `.nc` 样例文件，也可以直接运行这条命令。
依赖本地样例数据的补充校验测试会自动跳过，不会导致全量测试失败。

### 2. 开发模式启动

```powershell
mvn javafx:run
```

### 3. 打包为 Windows 安装程序

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\package-exe.ps1
```

打包完成后，安装包位于：

- `target\installer\NetCDFViewer-<version>.exe`

## 使用说明

### 打开数据

启动后可通过以下方式打开 NetCDF 文件：

- 菜单栏 `File -> Open...`
- 主界面按钮 `Open`
- 直接拖拽 `.nc` 文件到窗口

### 可视化流程

1. 打开一个或多个数据文件
2. 在左侧数据集列表中切换当前活动数据集
3. 在变量列表中选择可平面化变量
4. 若变量包含深度维，则使用右侧滑块切换层
5. 当数据中存在兼容的 `wdir` / `wlen` 或 `u` / `v` 变量对时，可按需启用波向箭头或海流流线叠加
6. 如需查看局部值，可在主图上点击进行单点查询
7. 如需导出，点击 `Export PNG`

## 常见问题

### 1. 为什么仓库里没有示例 `.nc` 文件？

因为示例文件体积较大，直接提交到公开仓库会让仓库变得臃肿，也不利于后续克隆和分发。因此默认通过 `.gitignore` 排除。
如果当前环境没有这些本地样例文件，相关补充测试会自动跳过，不影响公开仓库的基础测试通过。

### 2. 为什么不能直接双击 `jar` 运行？

本项目依赖 JavaFX 运行时，普通 `jar` 不是完整分发包。开发时请使用 `mvn javafx:run`，分发时请使用打包生成的 `.exe` 安装程序。

### 3. 导出的 PNG 如果异常怎么办？

当前版本已经加入导出后读回校验。如果导出失败，程序应直接报错而不是静默生成损坏文件。若仍有问题，请优先反馈运行环境、数据文件类型和操作步骤。

### 4. 是否支持所有 NetCDF 文件？

不支持。当前项目主要面向两类数据：

- 带有节点坐标和三角形连接关系的非结构化三角网 NetCDF
- 带有可识别横纵坐标基准的标准格网 NetCDF

对于缺少关键坐标、连接关系或水平基准的文件，程序只会显示元数据而无法平面渲染。
