# 底图图层设计

## 目标

为 NetCDF 可视化增加底图图层能力。第一版支持 XYZ 在线瓦片底图，底图固定绘制在图层最下方，NetCDF 标量场、风浪流叠加和海岸线继续绘制在底图上方。

## 范围

- 内置 OpenStreetMap 标准地图。
- 支持用户输入自定义 XYZ URL 模板。
- URL 模板使用 `{z}`、`{x}`、`{y}` 占位符。
- 支持显示/隐藏底图。
- 支持底图透明度。
- 支持本地磁盘缓存，减少重复下载。
- 仅对经纬度坐标数据启用底图。

## 非范围

- 不渲染真正的矢量瓦片。
- 不实现 WMS/WMTS 协议。
- 不做复杂投影转换。
- 不提供离线瓦片包管理。

## 架构

```text
[底图图层] ──┬── BasemapLayer 保存底图定义
             ├── BasemapTileMath 处理 Web Mercator 瓦片计算
             ├── BasemapTileProvider 获取瓦片图像
             ├── HttpBasemapTileProvider 下载并缓存瓦片
             └── BasemapRenderer 按当前视口绘制底图
```

## 渲染顺序

```text
[Canvas] ──┬── 1 底图瓦片
           ├── 2 NetCDF 标量底图
           ├── 3 海流/波浪/风场叠加
           ├── 4 海岸线
           └── 5 状态提示
```

## 坐标前提

当前项目的视口使用数据原始 X/Y 作为世界坐标。底图第一版只在空间域范围符合经纬度边界时启用：

- X 范围必须在 `[-180, 180]` 附近。
- Y 范围必须在 `[-90, 90]` 附近。
- 超出范围时不绘制底图，并在状态栏给出提示。

## 错误处理

- 单张瓦片下载失败时跳过该瓦片，不中断 NetCDF 渲染。
- 自定义 URL 模板缺少 `{z}`、`{x}`、`{y}` 时拒绝保存。
- 缓存目录不可写时仍尝试在线读取，失败则跳过瓦片。

## 测试

- 校验 Web Mercator 瓦片坐标和模板替换。
- 校验底图渲染器能把瓦片画到当前视口。
- 校验有底图时 NetCDF 渲染背景可透明。
- 校验主界面包含底图控件。
