# Checked Dataset Rendering Design

## Goal

让已加载数据列表同时承担两个职责：

- 选择某一行：切换当前活动数据集，用于变量列表、属性面板、色标、单点查询和矢量叠加控制。
- 勾选某一行：决定该数据集是否参与画布渲染。

新打开的 `.nc` 文件默认勾选并立即参与渲染。用户取消勾选后，数据仍保留在列表里，但从画布移除。

## Scope

Included:

- 已加载数据每行增加勾选框。
- 新打开数据集默认勾选。
- 多个勾选数据集按列表顺序叠加渲染到同一画布。
- 取消勾选后立即重绘，画布移除该数据集。
- 选择行仍切换当前活动数据集。
- 删除数据集时同步移除它的渲染勾选状态和缓存。
- 没有任何勾选数据集时显示提示，不清空已加载列表。
- 自动测试覆盖默认勾选、取消渲染、重新勾选、多数据叠加的状态流转。

Excluded for this iteration:

- 每个数据集独立选择变量。
- 每个数据集独立颜色表、透明度和数值范围。
- 多个数据集的波浪箭头、流线、风羽同时叠加。
- 图层拖拽排序。
- 图例同时展示多个数据集的色标。

## Chosen UX

左侧“已加载数据”列表保留单行选择能力，并在每行左侧增加勾选框。

Interaction rules:

- 打开文件成功后，列表新增一行并默认勾选。
- 勾选框控制渲染开关。
- 点击行本身切换当前活动数据集。
- 当前活动数据集驱动变量列表、摘要、属性、警告、坐标选择、色标和查询上下文。
- 多个勾选数据集叠加绘制时，底图只绘制一次，位于最底层。
- 标量图按列表顺序绘制，后面的数据覆盖前面的数据。
- 波浪箭头、流线、风羽只跟当前活动数据集走。

## Architecture

### Dataset render state

保留现有 `loadedDatasets` 和 `currentDataset` 模型，新增一个按源路径记录的渲染启用状态：

- `renderEnabledDatasetPaths`
- key 使用规范化后的绝对路径
- value 表示该数据集是否参与画布渲染

这样可以避免改动 `LoadedDatasetItem` 的不可变 record 结构，也能让删除、去重和缓存清理继续按源路径工作。

### Cell rendering

`MainController` 给 `datasetList` 设置自定义 cell：

- cell 内包含 `CheckBox` 和文件名文本。
- checkbox 的选中状态来自 `renderEnabledDatasetPaths`。
- 用户切换 checkbox 后更新状态并调用 `renderCurrentSelection()`。
- cell 复用时要解绑旧监听，避免滚动或刷新后误触发。

### Render pipeline

新增“可渲染数据集快照”概念：

- 从 `loadedDatasets` 中筛选已勾选条目。
- 每个条目解析出用于绘制的变量、空间域、层号和值数组。
- 活动数据集仍使用当前变量、当前层号和当前坐标绑定。
- 非活动数据集使用默认可绘制变量和第 0 层。

多数据渲染的第一版复用当前颜色表和范围策略：

- 色图使用当前全局颜色表。
- 自动范围按所有参与绘制的数据值合并计算。
- 手动范围继续使用现有输入框。

底图处理：

- 视口适配使用所有参与绘制数据集的空间范围合并结果。
- 底图只绘制一次。
- 如果参与绘制的数据集中存在非经纬度坐标，底图仍只对经纬度数据有效；非经纬度数据继续绘制在同一画布坐标系下。
- 状态栏提示跳过底图的原因，不中断渲染。

### Active dataset behavior

当前活动数据集不等同于参与渲染数据集。

- 选择行会改变活动数据集。
- 取消活动数据集的勾选后，它仍可显示变量和属性。
- 如果当前活动数据集未勾选，矢量叠加层不绘制。
- 点击查询优先返回当前活动数据集的查询结果。若活动数据集未勾选，则提示当前活动数据未参与渲染。

## Error Handling

- 单个已勾选数据集读取或绘制失败时，跳过该数据集，并在状态栏合并提示。
- 勾选的数据集中没有可绘制变量时，保留勾选状态，但渲染时跳过。
- 所有勾选数据集都无法绘制时，画布显示“已勾选数据没有可绘制变量”。
- 删除数据集时同步删除渲染启用状态和层缓存。
- 重复打开已加载文件时，不新增行；切换到已有行并保持原勾选状态。

## Testing Strategy

Add controller/UI tests covering:

- 新打开数据集默认勾选。
- 取消勾选后仍在列表中，但不参与渲染。
- 重新勾选后触发渲染。
- 多个数据集勾选状态可以同时为 true。
- 删除数据集会清理勾选状态。
- 取消所有勾选后不清空活动数据集面板。

Existing tests for single active dataset switching, invalid geographic coordinate warning, basemap skipping, PNG export, and overlays should continue to pass.

## Main Path

```text
[打开nc] ──┬── 加入已加载列表
           ├── 默认勾选参与渲染
           ├── 切换为当前活动数据
           └── 重绘画布 ──┬── 合并勾选数据范围
                          ├── 绘制底图
                          ├── 按列表顺序绘制标量场
                          └── 绘制当前活动数据的叠加层
```

## Review Notes

- 本设计刻意保留“一个当前活动数据集”。
- 多数据叠加先解决标量场同时可见。
- 每个数据集独立变量和样式属于下一阶段，不放进本次改动。
