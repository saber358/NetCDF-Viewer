# 更新日志

本文件记录项目的主要版本变化。

## 未发布

- 加载坐标变量看起来是经纬度、但 X/Y 范围不符合经纬度边界的 NetCDF 文件时，新增加载确认弹窗。
- 弹窗会展示文件名、坐标变量、合法范围、实际范围、异常原因和继续加载后的影响；用户取消时不加入数据列表。
- 保持底图安全策略不变：用户选择继续加载后，数据仍可渲染和查询，但在线底图会继续跳过。
- 更新 `README` 和回归测试，补充 `HYD.nc`、`DSD1211.nc` 与非经纬度命名坐标的覆盖。
- 重新打包 Windows 安装程序，更新 `Release/NetCDFViewer-1.1.4.exe`。

## 1.1.4

- 新增 XYZ 在线底图，内置 OpenStreetMap 标准地图，支持自定义 `{z}` / `{x}` / `{y}` 瓦片 URL 模板
- 底图固定绘制在 NetCDF 标量场下方，支持透明度调节和本地瓦片缓存
- 启用底图时标量渲染背景改为透明，避免数据背景遮挡瓦片
- 修复 Windows 用户代理未被 Java 自动读取时 OSM 瓦片直连超时的问题
- 修复全局视图下 OSM Web Mercator 瓦片按经纬度线性拉伸导致的底图错位
- 更新 `README`，补充在线底图叠加说明和实际渲染截图
- 调整打包脚本，补充 `java.net.http` 运行时模块，并将安装包放入 `Release`
- 补充底图瓦片数学、底图渲染、HTTP 瓦片下载、透明背景和界面控件回归测试
- 重新打包 Windows 安装程序，生成 `NetCDFViewer-1.1.4.exe`
- 版本号升级到 `1.1.4`

## 1.1.2

- 新增三角网空间索引，单点查询、流线积分、波浪箭头采样和风羽采样不再对全部三角形做线性扫描
- 渲染主链改为固定线程池调度，取消每次渲染临时创建后台线程的做法
- 底图、波浪、海流、风场叠加改为并行子任务构建，减少一次渲染中的串行等待
- 规则格网标量底图改为像素缓冲渲染，并支持按像素行并行栅格化
- 新增三角网空间索引测试与规则格网并行渲染一致性测试
- 更新 `README`，补充当前版本性能优化说明

- 版本号升级到 `1.1.2`

## 1.1.1

## 1.1.0

- 新增标准格网 NetCDF 支持，可加载规则格网类型数据，不再局限于非结构化三角网
- 新增统一空间域模型，三角网与标准格网共用主控制链路、视口适配和渲染入口
- 新增标准格网标量渲染，支持规则一维横纵坐标的节点场与单元场二维平面显示
- 新增标准格网单点查询，点击后可返回命中节点或单元的行列位置与数值
- 新增标准格网坐标基准切换，标准格网数据可按当前变量自动匹配兼容横纵坐标，并允许用户切换候选坐标轴
- 新增标准格网向量采样，支持同基准和交错基准的 `u` / `v` 结构化流场联合采样
- 新增标准格网海流流线叠加，支持结构化 ROMS / WRF 类速度场渲染动态流线
- 新增标准格网波向箭头采样，为后续结构化波场叠加提供统一采样基础
- 新增结构化渲染、点查、坐标控件、向量采样、流线与控制器回归测试，并完成 Maven 全量测试通过
- 版本号升级到 `1.1.0`

## 1.0.5
